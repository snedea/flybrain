## Phase 6: Educational 3D Brain Visualization

- [x] T6.1: Build interactive 3D Drosophila brain model with real-time connectome activity -- Add a toggleable full-screen panel containing a Three.js 3D brain visualization that shows the fly's neural activity in real time. The brain should be a stylized but anatomically-informed representation of the Drosophila brain with distinct volumes for each major neuropil region, colored by functional type (sensory blue, central purple, drives amber, motor red). Each region's opacity/emissive glow should be driven by the live connectome activation levels so users can watch the brain "light up" as the fly behaves. Implementation: (1) Add Three.js via CDN (no build step) in index.html, (2) Create js/brain3d.js with a Brain3D module that initializes a WebGL scene, camera (perspective with orbit controls via Three.js OrbitControls), ambient + point lighting, and a renderer sized to its container, (3) Model the brain regions as positioned/scaled meshes using Three.js built-in geometries (SphereGeometry, ellipsoids via scaled spheres, TorusGeometry for mushroom body lobes) arranged in anatomically approximate positions: optic lobes (VIS_*) as two large lateral hemispheres, antennal lobes (OLF_*) as two smaller anterior spheres, mushroom bodies (MB_*) as distinctive paired peduncle+calyx+lobe structures in the dorsal-posterior brain, central complex (CX_*) as a midline disc/fan structure, lateral horn (LH_*) as paired lateral volumes, subesophageal zone (SEZ_*, GNG_*) as a ventral mass, ventral nerve cord (VNC_*, DN_*, MN_*) as an elongated ventral cylinder, (4) Use MeshPhongMaterial or MeshStandardMaterial with transparency (opacity: 0.3 base) and emissive color; each mesh gets an update() method that reads the relevant BRAIN.postSynaptic neuron states, averages them, and maps to emissive intensity (0.0-1.0) and opacity (0.3-0.8), (5) Add a toggle button to the toolbar ("Brain 3D") that shows/hides the 3D panel as an overlay, (6) Add hover detection via raycasting: when the user hovers a region, show a tooltip with the region name, description, and list of neuron groups it contains with their current activation levels, (7) Render loop calls Brain3D.update() each frame, reading from BRAIN.postSynaptic to update all region glows. Region-to-neuron mapping: Optic Lobes = [VIS_R1R6, VIS_R7R8, VIS_ME, VIS_LO, VIS_LC, VIS_LPTC], Antennal Lobes = [OLF_ORN_FOOD, OLF_ORN_DANGER, OLF_LN, OLF_PN], Mushroom Bodies = [MB_KC, MB_APL, MB_MBON_APP, MB_MBON_AV, MB_DAN_REW, MB_DAN_PUN], Central Complex = [CX_EPG, CX_PFN, CX_FC, CX_HDELTA], Lateral Horn = [LH_APP, LH_AV], SEZ = [SEZ_FEED, SEZ_GROOM, SEZ_WATER, GUS_GRN_SWEET, GUS_GRN_BITTER, GUS_GRN_WATER], VNC/Motor = [DN_WALK, DN_FLIGHT, DN_TURN, DN_BACKUP, DN_STARTLE, VNC_CPG, MN_*], Thermosensory = [THERMO_WARM, THERMO_COOL], Mechanosensory = [MECH_BRISTLE, MECH_JO, MECH_CHORD, ANTENNAL_MECH], Drives = [DRIVE_HUNGER, DRIVE_FEAR, DRIVE_FATIGUE, DRIVE_CURIOSITY, DRIVE_GROOM]. Files: index.html, js/brain3d.js (new), css/main.css. [SPI-]

## Discovery Round 18

- [x] D18.1: Fix education panel highlightRegion being invisible because Brain3D.update() overwrites mesh material values every frame -- The highlightRegion() function at education.js:248-278 saves current mesh opacity/emissiveIntensity, boosts them to 1.5/0.9, and uses setTimeout to restore after 1200ms. But Brain3D.update() at brain3d.js:303-328 is called every animation frame from main.js:1769, and it recalculates and overwrites opacity and emissiveIntensity for every region mesh based on activation data. The highlight is visible for at most one frame (~16ms) before update() stomps on it, making the feature completely non-functional. The setTimeout restore at 1200ms also writes back stale values captured before the highlight, causing a brief flicker to outdated opacity/emissive levels. Fix: add a per-region _highlightUntil timestamp or boolean flag checked by Brain3D.update() so it skips recalculating material values for a region that is currently highlighted; highlightRegion() sets the flag with a 1200ms expiry, and update() respects it by leaving that region's meshes untouched until the flag expires. Files: js/brain3d.js, js/education.js. [-PI-]

- [x] D18.2: Fix neuron-to-region classification mismatches between connectome dot panel and Brain3D/education panel, and add 3 orphaned neuron groups to 3D brain and education -- Two related data-consistency issues between the connectome dot panel (uses BRAIN.neuronRegions from connectome.js:100-128) and the Brain3D/education modules (brain3d.js REGION_DEFS, education.js EDUCATION_REGIONS): (1) Three neuron groups exist in the connectome, have weights, fire during simulation, and appear in neuronPopulations, but are completely absent from all Brain3D regions and education panel sections: NOCI (nociception/pain, connectome.js:107, fires on rapid touch per connectome.js:355), GNG_DESC (gnathal ganglia descending, connectome.js:114), and CLOCK_DN (circadian clock descending, connectome.js:116); the education panel intro says "59 functional neuron groups" but only describes 56 across its 10 region sections; add NOCI to the Mechanosensory region (it is a sensory pain pathway), GNG_DESC to the Subesophageal Zone region (gnathal ganglia is part of SEZ), and CLOCK_DN to the Central Complex or Drives region (circadian modulation), (2) Several neurons have conflicting type classifications causing visible color mismatches between the connectome dots (purple/central) and Brain3D/education (different colors): ANTENNAL_MECH is central in neuronRegions (connectome.js:114) but sensory in brain3d.js:104 and education.js:87; DN_WALK, DN_FLIGHT, DN_TURN, DN_BACKUP, DN_STARTLE, and VNC_CPG are central in neuronRegions (connectome.js:115-116) but motor in brain3d.js:85-86 and education.js:66; users viewing both panels simultaneously see contradictory color coding for the same neurons; update BRAIN.neuronRegions to match the brain3d/education categorization which is more neuroscientifically accurate (ANTENNAL_MECH is sensory input, DN_*/VNC_CPG are motor pathway). Files: js/connectome.js, js/brain3d.js, js/education.js. [-PI-]
